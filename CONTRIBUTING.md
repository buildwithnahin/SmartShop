# Contributing Guide

Thank you for contributing to SmartShop.

## Setup

1. Fork the repository
2. Create a feature branch
3. Follow coding and commit conventions
4. Open a pull request

## Development Rules

- Follow project architecture and module boundaries
- Keep functions focused and testable
- Add validation and error handling for new endpoints
- Update docs for any API or schema change

## Commit Style

Use Conventional Commits.

## Pull Request Checklist

- [ ] Code compiles and runs
- [ ] Tests added or updated
- [ ] Docs updated
- [ ] No secrets committed
